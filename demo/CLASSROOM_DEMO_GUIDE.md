# 🎓 课堂演示指南 | Classroom Demo Guide

## 🔥 推荐方案：热点捕获模式

这是最佳的课堂互动方案！让同学连接你的热点，你就能捕获他们的流量。

### 快速开始

```bash
cd /Users/josiezhou/Documents/IoT-device-classif
./demo/hotspot_capture.sh
```

### 详细步骤

#### 步骤 1: 开启 Mac 热点

1. 打开 **系统设置** → **通用** → **共享**
2. 点击 **互联网共享**
3. "共享以下来源的连接": 选择 **以太网** 或 **USB iPhone**（如果用以太网上网）
4. "用以下端口共享给电脑": 勾选 **Wi-Fi**
5. 点击 **Wi-Fi 选项**，设置：
   - 网络名称：如 `IoT_Demo`
   - 密码：设置一个简单的密码告诉同学
6. 打开 **互联网共享** 开关

#### 步骤 2: 让同学连接

告诉同学连接你的热点 WiFi，然后正常使用手机上网。

#### 步骤 3: 运行捕获脚本

```bash
./demo/hotspot_capture.sh
```

#### 步骤 4: 上传分析

打开 Demo 上传生成的 pcap 文件，展示分类结果！

## 备选方案：直接捕获（同一网络）

> ⚠️ 注意：在学校/企业网络中通常无法捕获其他设备的流量

### 1️⃣ 让同学查看自己的 IP 地址

**iPhone/iPad:**
```
设置 → Wi-Fi → 点击已连接的网络 → 查看 IP 地址
```

**Android:**
```
设置 → 关于手机 → 状态信息 → IP 地址
或
设置 → Wi-Fi → 点击已连接的网络
```

**Windows:**
```
打开 CMD，输入: ipconfig
查看 IPv4 地址
```

**Mac:**
```
打开终端，输入: ifconfig en0 | grep inet
或
系统偏好设置 → 网络 → Wi-Fi → 查看 IP
```

## 常见问题

### Q: 如果捕获不到数据包怎么办？
A: 确保你和同学在同一个 WiFi 网络，某些网络可能有隔离策略。

### Q: 为什么有些设备识别不准？
A: 模型对 PC/smartphone/Audio 等设备识别率最高（>95%），
   对 Lighting/Motion_Sensor 等识别率较低，这是数据不平衡导致的。

### Q: 需要多少数据包才能分析？
A: 通常 100-500 个数据包就足够了。

## 技术要点（可在演示中提及）

- **模型**: XGBoost，准确率 88.78%
- **数据集**: UNSW HomeNet，93万+样本
- **特征**: 75个网络流量特征（端口、包大小、时间间隔等）
- **设备类型**: 14种（手机、电脑、摄像头、路由器等）
